# Системное ПО

1.Операционные системы(ОС).
2.Системы управления файлами.
3.Интерфейсные оболочками.
4.Системы программирования.
5.Утилиты.

## Определение понятия "операционная система"
*Под ОС обычно понимают комплекс взаимносвязанных управляюжщих и обрабатывающих программ,которые,с одной стороны,организуют интерфейс между аппаратурой комьютера и пользователем и запущенными приложениями,а с другой стороны-предназначены для организации наиболее эффективного использования ресурсов компьютера и организации выполнения программ.*

**Система управления файлами**-предназначена для организации более удобного досуга к данным,организованным в виде файлов,т.е вместо низкоуровневого доступа через физического адреса используетя логический путь с указанием полного пути и имени файла.

**Интерефейсные оболочки**-служат для взаимодействия ОС с конечным пользователем через терминал или GUI.

**Системы программирования**-используется для разработки софта.

**Утилиты** используются для обслуживания ОС,а также для подготовки к работе носителей данных,оптимизацию и.т.д

# История появления оперционных систем
*Эволюция системного ПО и ОС,в частности,напрямую связана с эволюцией аппратных средст вычислительной техники,а именно с появлением новой элементной базы,на которой строилось каждое новое поколение комьютеров.*

*Рассмотрим более подробно этапы становления и развития техники и системного ПО.*
# Появление первых ОС
- 1940-е Созданы первые ламповые ЭВМ,на перфокартах(нулевое поколение)

- **| поколение ОС** (использование полупроводников,развитие языков программирования) середина 1950-х
- начало 1960-х **||поколение ОС** (использование процессорной обработки данных,интегрельных схемы)
- **||| поколение ОС** (программная прослойка между ЭВМ и пользователем).Середина 1960-х-1970-е
# Нулевое поколение ЭВМ
*В эти годы компьютеры представляли собой гигансткие машины,которые выполняли программы с помощью перфокарт и перфолент.Управление ими было минимальным,программирование происходило напрямую н аппаратном уровне,что требовало больших усилий и времени.*
# Перове поколение ЭВМ
- *Основная задача ОС 50-х годов-упрощение прехода с задачи на задачу (ОС пакетной обработки)*.
- *Запущенная задача получала в свое распоряжение все ресурсы компьютера.*
- После завершения каждой задачи (нормальное или аврарийного)управление ресурсами возращалось ОС,которая приводила ЭВМ в соствояние,позволяющее ввести и запустить следующую задачу,и обеспечивала ввод и запуск этой задачи.
- **Главная цель такой ОС**-сокращение времени на запуск программы и удаление ее из машины.
# Третье поколение ЭВМ
- Операционные систем третьего поколения стали программной прослойкой между пользователями и аппаратурой ЭВМ.
- Они являются многорежимными,обеспечивающими обработку информации во всех известных режимах:
   - Пакетную обработку;
   - Разделение времени;
   - Режим реального времени;
   - Мультипроцессорный режим.
   - Универсальность этих систем обусловила их громоздкость и дороговизну.
- Для работы с такими системами пользователю приходилось изучать сложными языки управления заданиями,чтобы уметь описывать задания и требуемые ресурсы(UNIX).

# Скрипты и автоматизация терминала
- Копирование файла:`copy`<путь исходним> <путь копии>
- Перемещение файла:`move data.txt` `C:\data_2`
- Удалить папку:`rmdir` `C:\data_2`
Если папка не пуста:`rmdir/s C:\data_2`
- В терминале можно использовать циклическую конструкцию:` for in (1,2,3) do echo`
- Оператор ветвления тоже присктсвует в терминале,например `if not exits "data.txt"echo not found`или `if exist "data.txt" echo found`
# А вот теперь напишем скрипты
- Чтобы каждый раз не писать последовательность команд,их можно вынести в отдельных скрипт.Для этого в папке напишем команду `notepad`<name of script>.bat
- Теперь в этом скрипте можно написать последовательности команд с каждой новой строчки.
- Давайте запустим калькулятор и блокнот,напишем такой скрипт
`@echo off`
`start calc.exe`
`start notepad.exe`
- Сохраним и запустим в консоли как <name of script>.bat
- Для написания комментариев используем команду `rem`
# Скрипт:задачи
- Создайте скрипт `script.bat`и пропишите в нём команды,которые создают папку `date_<текущая дата>на диске C:\
- Создайте 5 текстовых файлов `text`_<номер>.txt с помощью цикла с содержимым "l'm guru of cmd-script_<номер файла>".Для этого можно воспользоваться командой `echo my text to script`>file.txt
- Выведите файл `python` с содержимым `print`("Задача:",2+2*2)
- Затем запустите этот файл на исполнение.
- **Примечание**: перед переменной в скриптах ставится %%i вместо %i
# Скрипт:задача
```bash
@echo off
rmdir /s /q date_ _%date%
mdkdir date %date%
cd %%i in (1,2,3,4,5)do echo I`m guru of cmd-script_%%i > text_%%i.txt
for %%i > text_%%i.txt
for %%i in (1,2,3,4,5) do type text_%%i.txt
echo prtint ("Задача", 2+2*2) > task.py
python task.py
cd C:\data
```
