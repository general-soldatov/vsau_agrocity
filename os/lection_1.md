# Системное ПО
К системному ПО (System Software) относятся программы и комплексы программ, являющимися общими для пользователей, кто совместно использует технические средства компьютера, и применяемые как для автоматизации разработки новых программ, так и для организации процесса выполнения существующих программ.
Системное ПО можно разделить на группы:


1.  Операционные системы (ОС).
2.  Системы управления файлами.
3.  Интерфейсные оболочки.
4.  Системы программирования.
5.  Утилиты. 

 Теперь рассмотрим подробно понятие ОС.

# Определение понятия «операционная система»
Под ОС обычно понимают комплекс взаимосвязанных управляющих и обрабатывающих программ, которые, с одной стороны, организуют интерфейс между аппаратурой компьютера и пользователем и запущенными приложениями, а с другой стороны – предназначены для организации наиболее эффективного использования ресурсов компьютера и организации выполнения программ.

**Система управления файлами** предназначена для организации более удобного доступа к данным, организованным в виде файлов, т.е. вместо низкоуровневого доступа через физические адреса используется логический путь  с указанием полного пути и имени файла.

**Интерфейсные оболочки**  служат для взаимодействия ОС с конечным пользователем через терминал или GUI.

**Системы программирования** используются для разработки софта.

**Утилиты** используются для обслуживания ОС, а также для подготовки к работе носителей данных, оптимизацию и т.д.

# История появления операционных систем
Эволюция системного ПО и ОС, в частности, напрямую связана с эволюцией аппаратных средств вычислительной техники, а именно с появлением новой элементной базы, на которой строилось каждое новое поколение компьютеров.

Рассмотрим более подробно этапы становления и развития техники и системного ПО.
# Появление первых ОС
- **1940-е** Созданы первые ламповые ЭВМ, на перфокартах **(нулевое поколение)**
- **середина 1950-х** **I поколение ОС** (использование полупроводников, развитие языков программирования)
- **начало 1960-х** **II поколение ОС** (использование процессорной обработки данных, интегральные схемы)
- **Середина 1960-х - 70-е** **III поколение ОС** (программная прослойка между ЭВМ и пользователем)
- **середина 1970-х – 90-е** **IV поколение ОС** (использование микропроцессоров, появление ПК)
# Нулевое поколение ЭВМ
В эти годы компьютеры представляли собой гигантские машины, которые выполняли программы с помощью перфокарт и перфолент. Управление ими было минимальным, программирование происходило напрямую на аппаратном уровне, что требовало больших усилий и времени.
# Первое поколение ЭВМ
- Основная задача ОС 50-х годов — упрощение перехода с задачи на задачу (ОС пакетной обработки).
- Запущенная задача получала в свое распоряжение все ресурсы компьютера.
- После завершения каждой задачи (нормального или аварийного) управление ресурсами возвращалось ОС, которая приводила ЭВМ в состояние, позволяющее ввести и запустить следующую задачу, и обеспечивала ввод и запуск этой задачи.
- **Главная цель такой ОС** — сокращение времени на запуск программы и удаление ее из машины.
# Второе поколение ЭВМ
- Начали появляться методы, обеспечивающие независимость программирования от внешних устройств.
- В системах второго поколения было использовано мультипрограммирование для устранения недостатка, связанного с тратой основного времени на I/O, и, как правило, повышение эффективности использования ЦП.
- ОС второго поколения можно подразделить на 
    - Системы разделения времени пользователь непосредственно взаимодействует с компьютером при помощи терминалов, в интерактивном режиме. 
    - Системы реального времени часто работают с недогрузкой, они находятся в состоянии постоянной готовности к предусмотренным событиям.
# Третье поколение ЭВМ
- Операционные системы третьего поколения стали программной прослойкой между пользователями и аппаратурой ЭВМ.
- Они являются многорежимными, обеспечивающими обработку информации во всех известных режимах:
    - Пакетную обработку;
    - Разделение времени;
    - Режим реального времени;
    - Мультипроцессорный режим.
    - Универсальность этих систем обусловила их громоздкость и дороговизну.
- Для работы с такими системами пользователю приходилось изучать сложные языки управления заданиями, чтобы уметь описывать задания и требуемые ресурсы (UNIX).

# Четвёртое поколение ЭВМ
- **Появление четвертого поколения ОС связано с:**
    - распространением вычислительных сетей;
    - появлением микропроцессора и персонального компьютера.
- ПК оснащаются интерфейсными средствами приема-передачи данных и могут использоваться в качестве терминалов мощных ВС.
- Нет необходимости взаимодействовать с одним ПК — можно обращаться к территориально распределенным машинам ВС. При этом усложнились проблемы защиты информации от возможного несанкционированного доступа.
- **Также для IV поколения характерно:**
    - дружественный интерфейс
    - использование концепции виртуальных машин, благодаря которой пользователь избавлен от необходимости знать физические особенности машин и систем;
    - Распределенная обработка данных: гораздо целесообразнее иметь вычислительные мощности там, где они необходимы, вместо того, чтобы передавать данные для обработки в вычислительные центры.
# Современный этап развития операционных систем
- К началу 90-х гг практически все ОС стали сетевыми и сетевые функции встраиваются в ядро ОС.
- Появились специализированные ОС, которые предназначены исключительно для выполнения коммуникационных задач. 
- Во второй половине 90-х гг все производители ОС резко усилили поддержку средств работы с Интернетом. В результате ПК превратился из чисто вычислительного устройства в средство коммуникаций с развитыми вычислительными возможностями.
- Особое внимание уделялось корпоративным сетевым ОС, которая отличается способностью хорошо и устойчиво работать в крупных сетях.
- Очень велика роль единой справочной службы, важное значение имеют средства обеспечения безопасности ОС.
- Современная ОС наряду с выполнением основных функций должна поддерживать:
    - мультипрограммную обработку,
    - виртуальную память,
    - многооконный графический интерфейс пользователя
    - и выполнять многие другие функции..

Большое внимание уделяется повышению удобства работы человека.

# Семейство Windows
Компания Microsoft была создана в 1975 г., несмотря на то, что её первым программным продуктом был интерпретатор языка BASIC, в 1981 г. они выпустили первую ОС MS-DOS. 
Первый продукт Windows стал реализацией GUI над MS-DOS, а вот Windows 95 – стала полноценной ОС, реализующей GUI. 
В ней же появился API Win32, Plug&Play для устройств и появление кнопки «Пуск».
Отдельно развивалась линейка Windows NT (New Tech), оптимизированная не только под рабочие станции, но и серверные платформы.
В настоящее время Windows – одна из самых распространённых ОС, однако и популярны вредоносные программы.

# ОС, построенные на принципах UNIX
OC UNIX как таковая начала появляться в 1960-х годах, над которой работали специалисты Bell Labs, и в большой степени – Кеннет Лейн Томсон.
Рост популярности ОС связан с её поставкой многим университетам и открытому исходному коду, что позволило университетскому сообществу не только изучать, но и редактировать исходный код (open source).
Перенос кода ОС на язык Си позволил повысить универсальность её внедрения на разные ЭВМ, однако организационные нюансы правообладания способствовали снижению качества продукта и падению спроса.
# Операционная система Linux
- ОС Linux была создана в 1991 г. молодым студентом Линусом Торвальдсом. Хотя было много заимствований из ОС UNIX, новая ОС стала принципиально иной и имела монолитную структуру. 
- Благодаря успешности нового продукта, его развитие сопровождалось переносом большого количества функций из OC UNIX.
- Бизнес-модель системы не является стандартной и имеет лицензию GNU GPL  - общедоступная лицензия GNU, т.е. лицензируется как свободное ПО.
- В настоящее время имеет большую популярность в качестве серверных ОС. А также имеет множество дистрибутивов для рабочих станций.

# Red Hat Enterprise Linux
Red Hat Enterprise Linux — это операционная система, которая более 30 лет входит в топ-10 самых популярных серверных ОС. RHEL популярен благодаря своей надежности и стабильности,.
Так, летом 1995 года появилась первая версия Red Hat Linux с кодовым именем Mother’s Day.
Используется RHEL в основном для сервисов и инфраструктур, где критически важным фактором является отказоустойчивость и стабильность операционной системы. Этим обусловлен столь длинный срок поддержки каждой версии ОС — 10 лет с возможностью продления до 13.

# Debian
**Debian** — это ОС с открытым исходным кодом, которая состоит из свободного программного обеспечения. Она стала основой для многих других дистрибутивов, например, **Ubuntu**. Проект **Debian** начал разрабатываться в августе 1993 года.

**Debian-проект** является некоммерческим: он основан на принципе Free Software, т.е. может разрабатываться добровольцами со всего мира.

**Debian** — первая ОС, у которой появилась автоматическая система управления программным обеспечением под названием Advanced Packaging Tool **(APT)**. Благодаря ей достаточно просто сказать ОС, какое приложение нужно, и он автоматически найдёт как приложение, так и все необходимые для установки компоненты.

# Ubuntu
**Ubuntu** Linux — это ОС для ПК, которую часто встретите на компьютерах программистов и системных администраторов.

**Ubuntu** — это один из дистрибутивов Линукса, ориентированный на простых людей, которым нужно делать простые людские дела. Там есть графический интерфейс и офисные программы.
Ubuntu защищена на архитектурном уровне, поэтому может работать без антивируса. При этом новейшая версия операционной системы запустится на любом компьютере.
Ubuntu и весь сопутствующий софт — это бесплатный продукт, который для обычных пользователей останется бесплатным.
# FreeBSD
FreeBSD – это динамично развивающаяся операционная система с открытым кодом, удерживающая свою позицию в течение 30 лет.

Первый официальный релиз FreeBSD 1.0 состоялся 1 ноября 1993 года.
Одним из главных преимуществ FreeBSD является ее высокая стабильность и надежность. FreeBSD способна обрабатывать большие объемы данных и трафика, поддерживать высокую нагрузку и отказоустойчивость.
Одним из недостатков FreeBSD является ее сложность использования для новичков. FreeBSD требует от пользователя определенных знаний и навыков для установки, настройки и обслуживания системы

История Android началась в 2003 году, когда группа разработчиков во главе с Энди Рубином решила создать операционную систему для цифровых камер. Однако они быстро осознали потенциал мобильной ОС, учитывая стремительный рост рынка смартфонов. 
В 2005 году компанию Android Inc. приобрела Google, и это стало началом развития Android в том виде, в котором мы знаем её сегодня.

# Работа с файловой системой
- Чтобы открыть командную строку, наберите команду Win+R и в строке введите `cmd`.
- Для того, чтобы перейти к диску С:\ наберём команду: cd C:\
- Посмотрим содержимое папки: `dir`
- Создадим папку командой: `md` или `mkdir` `<название_папки>`
- Теперь научимся работать с текстовыми файлами. Чтобы создать файл, можно воспользоваться блокнотом: `notepad test.txt`.
- После этого откроется блокнот с новым текстовым файлом.
- Удалить этот файл можно командой `del` `test.txt`
# Работа с файловой системой: Задание
- Теперь мы можем создать файл с исходным текстом программы на `python`. Для этого наберём такую команду: `notepad test.py`.
- В новом окне напишем простую программу, например,`print("Hello, World!")`
- Затем, в окне терминала запустим программу с помощью встроенного интерпретатора: `python` `test.py`
- Посмотрите на результат выполнения программы.
# Работа с файловой системой
- Однако текст файла можно написать и без использования блокнота. Для этого применим команду: `copy con` `<название_файла>`.
- Затем в интерактивном режиме введите текст, который вы хотите добавить в файл, а после зажмите клавиши **Ctrl+Z**
- Просмотреть содержимое файла в терминале можно с помощью команды: `more` `<название_файла>` или аналогично `type`

# Скрипты и автоматизация терминала
- Копирование файла: `copy` `<путь_исходник>` `<путь_копии>`
- Перемещение файла: `move` `data.txt` `C:\data_2`
- Удалить папку: `rmdir` `C:\data_2`
- Если папка не пуста: `rmdir` `/s C:\data_2`
- В терминале можно использовать циклическую конструкцию:`for` `%i in (1, 2, 3) do echo %i`
- Оператор ветвления тоже присутствует в терминале, например `if not exist “data.txt” echo not found` или `if exist “data.txt” echo found`
# А вот теперь напишем скрипты
- Чтобы каждый раз не писать последовательность команд, их можно вынести в отдельный скрипт. Для этого в папке напишем команду `notepad <name_of_script>.bat`
- Теперь в этом скрипте можно написать последовательность команд с каждой новой строчки. 
- Давайте запустим калькулятор и блокнот, напишем такой скрипт:
```bash
@echo off
start calc.exe
start notepad.exe
```
- Сохраним и запустим в консоли как `<name_of_script>.bat`
- Для написания комментариев используем команду `rem`
# Скрипт: задача
- Создайте скрипт `script.bat` и пропишите в нём команды, которые создают папку `date_<текущая_дата>` на диске C:
- Создайте 5 текстовых файлов `text_<номер>.txt` с помощью цикла с содержимым “I'm guru of cmd-script_<номер файла>”. Для этого можно воспользоваться командой `echo` my text to script > `file.txt`
- Выведите содержимое этих файлов в консоль по очереди.
- Создайте файл `python`с содержимым `print("Задача:", 2 + 2 * 2)`
- Затем запустите этот файл на исполнение.
- **Примечание:** перед переменной в скриптах ставится %%i вместо %i
# Скрипт: задача
- Пример рабочего скрипта:
```bash
@echo off 
rmdir /s /q date_%date%
mkdir date_%date%
cd date_%date%
for %%i in (1, 2, 3, 4, 5) do echo I`m guru of cmd-script_%%i > text_%%i.txt
for %%i in (1, 2, 3, 4, 5) do type text_%%i.txt
echo print("Задача", 2 + 2 * 2) > task.py
python task.py
cd C:\data
```
