# Требования к современным ОС
### ОС является сердцевиной ПО компьютера, она создаёт среду для выполнения приложений. Одно из главных требований – выполнение основных функций.
- Эффективное управление ресурсами.
- Удобный интерфейс для пользователя и прикладных программ.
- Соответственно, должна быть поддержка:
    - многозадачности
    - виртуальной памяти
    - многооконного GUI
# Требования к современным ОС
### Рассмотрим теперь эксплуатационные требования к ОС
- **Расширяемость:** код может дополняться и изменяться без нарушения целостности.
- **Переносимость:** поддержка различных аппаратных платформ.
- **Совместимость:** возможность запуска приложений, написанных для других ОС.
- **Надёжность и отказоустойчивость:** защита от внутренних и внешних ошибок, сбоев и отказов.
- **Безопасность:** предотвращение несанкционированного доступа к данным.
- **Производительность:** оптимальное время быстродействия и времени реакции для аппаратного обеспечения
# Классификация ОС
### 1. По назначению
- **Системы общего назначения:** ОС для широкого круга задач, в т.ч. запуск различных приложений, их разработка, работа с сетью.
- **Системы реального времени:** ОС для управления летательными аппаратами, автомобилями, технологическими установками.
- **Прочие специализированные системы:** направлены на решение узконаправленных задач, например, сетевые системы типа Novell Netware, обеспечивающие надёжное функционирование локальных сетей.
# Классификация ОС
### 2. По характеру взаимодействия с пользователем
- **Пакетные:** обрабатывающие заранее подготовленные задания.
- **Диалоговые:** выполняющие команды пользователя в интерактивном режиме.
- **ОС с графическим интерфейсом:** разновидность диалоговых систем, с использованием графических манипуляторов (мыши), т.е. со своей спецификой.
- **Встроенные ОС:** не взаимодействующие с пользователем
# Классификация ОС
### 3. По числу одновременно выполняемых задач
- **Однозадачные:** в таких системах в каждый момент времени может существовать не более чем один активный пользовательский процесс.
- **Многозадачные:** обеспечивают параллельное выполнение нескольких пользовательских процессов. Характеризуется значительным усложнением алгоритмов и структур данных, используемых в системе.
# Классификация ОС
### 4. По числу пользователей.
- **Однопользовательские:** для них характере полный доступ пользователя к ресурсам системы. Подобные системы приемлемы в основном для изолированных компьютеров, не допускающих дом=ступа по сети или удалённых терминалов.
- **Многопользовательские:** должна обеспечивается защита данных и процессов каждого пользователя, точное указание прав доступа, предоставленных каждому пользователю системы.

## Внутреннее строение ОС: монолитное ядро
- Монолитное ядро = ОС
- Точки входа в систему – системные вызовы
- Абстракция для системных вызовов
- Пользовательские программы
    - Каждая процедура может вызывать каждую.
    - Все процедуры работают в привилегированном режиме.
    - Ядро совпадает со всей операционной системой.
    - Пользовательские программы взаимодействуют через системные вызовы.
# Внутреннее строение ОС: Многоуровневые (Layered) системы
0. Hardware
1. Планирование задач и процессов 
2. Управление памятью
3. Драйвер связи с консолью
4. Управление I/O
5. Интерфейс пользователя 
-  Процедуры уровня К могут вызывать только процедуры К – 1.
-  Все или почти все уровни могут быть вызваны в привилегированном режиме.
- Ядро совпадает или почти совпадает со всей операционной системой.
- Систему легко отлаживать, поскольку ошибки не переходят на другой уровень.
- Пользовательские программы будут взаимодействовать только через интерфейс пользователя.
# Внутреннее строение ОС: микроядерная (microkernel) архитектура
### Микроядро
Функции микроядра:
- Взаимодействие между программами.
- Планирование использование процессора
- Первичная обработка прерываний и операций I/O, базовое управление памятью.

# Все представленные архитектуры ОС не лишены недостатков
### Поэтому найти коммерческую ОС, работающую по той или иной архитектуре – невозможно.
# Внутреннее строение ОС
### Смешанные системы – почему?
- Монолитное ядро – необходимость перекомпиляции при каждом изменении, сложность отладки, высокая скорость работы.
- Многоуровневые системы – необходимость перекомпиляции при изменениях, отлаживается только изменённый уровень, меньшая скорость работы.
- Микроядро – простота отладки, возможность замены компонент без перекомпиляции и остановки системы, очень медленные.
# Linux Ubuntu
Linux представляет собой систему с монолитным ядром, однако имеющую черты, перенесённые из микроядерной архитектуры с некоторыми видоизменениями.
То есть при компиляции ОС можно не включать некоторые компоненты, некоторые модули оставить подгружаемыми с хранением на жёстком диске, но работающими в привилегированном режиме.
